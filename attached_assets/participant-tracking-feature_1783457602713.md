## 1. نظرة عامة على الميزة

### الهدف
تمكين مدير المشروع من تحديد قائمة بالمشاركين المطلوب منهم تعبئة النموذج، ومتابعة حالة كل منهم في الوقت الفعلي، مع إرسال تذكيرات مستهدفة ذكية للمتأخرين عبر تيليغرام.

### الفجوة الحالية
النظام الحالي يعرف **من عبّأ فعلاً**، لكنه لا يعرف **من كان مفترضاً أن يعبّئ**. هذه الميزة تُغلق هذه الفجوة وتُتيح قياس نسبة الاستجابة الحقيقية.

### أمثلة حالات الاستخدام
- إدارة موارد بشرية تتابع من أكمل نموذج الإجازة من 150 موظفاً
- مدرسة تتابع من أعاد نموذج موافقة ولي الأمر من 200 طالب
- جهة حكومية تتابع إقرارات 80 موظفاً

## 2. آلية العمل الكاملة

```
1. المدير يُفعِّل "تتبع المشاركين" في إعدادات المشروع
          ↓
2. الرابط العام يُغلق تلقائياً لهذا المشروع فقط
   (يظهر لمن يفتحه: "هذا النموذج مخصص للمدعوين فقط")
   (خيار: السماح لغير المُسجَّلين يُفعَّل يدوياً عند الحاجة)
          ↓
3. المدير يُضيف المشاركين (يدوياً أو باستيراد Excel)
          ↓
4. النظام يُولّد token فريد لكل مشارك ويُرسل تيليغرام تلقائياً
   الرسالة تحتوي: الرابط الشخصي + رابط تفعيل البوت
          ↓
5. المشارك يفتح رابط البوت مرة واحدة → النظام يحفظ chat_id تلقائياً
          ↓
6. المشارك يفتح رابطه الشخصي → يُسجَّل first_opened_at
   النموذج يظهر مع بياناته مُعبّأةً مسبقاً (بدون كود دعوة)
          ↓
7. بعد التعبئة → يُربط السجل بالمشارك
   يمكن التعديل خلال participant_edit_hours
   بعدها: رسالة "انتهت فترة التعديل، شكراً لك"
          ↓
8. المدير يُتابع لوحة التحكم: 3 حالات في الوقت الفعلي
   ويُرسل تذكيرات ذكية للمتأخرين
```

## 3. تصميم قاعدة البيانات

### الجدول الجديد: `project_participants`

```sql
CREATE TABLE project_participants (
  id                UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id        UUID        NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  name              TEXT        NOT NULL,
  identifier        TEXT,                      -- رقم جوال أو إيميل (اختياري)
  identifier_type   TEXT,                      -- 'phone' | 'email' | NULL
  token             UUID        NOT NULL UNIQUE DEFAULT gen_random_uuid(),
  telegram_chat_id  TEXT,                      -- يُحفظ بعد أن يفتح المشارك رابط البوت
  prefill_data      JSONB,                     -- بيانات التعبئة المسبقة من استيراد Excel
  record_id         UUID        REFERENCES project_records(id) ON DELETE SET NULL,
  submitted_at      TIMESTAMPTZ,               -- يُعيَّن عند ربط السجل — لا يُحذف عند حذف السجل
  first_opened_at   TIMESTAMPTZ,              -- أول مرة يفتح الرابط (لم يكمل بعد)
  last_notified_at  TIMESTAMPTZ,
  notify_count      INT         NOT NULL DEFAULT 0,
  added_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
  notes             TEXT
);

CREATE INDEX ON project_participants(project_id);
CREATE INDEX ON project_participants(token);
CREATE INDEX ON project_participants(record_id);
CREATE INDEX ON project_participants(telegram_chat_id);
```

> **ملاحظة مهمة:** `submitted_at` يُخزَّن مستقلاً عن `record_id`. إذا حُذف السجل لاحقاً فإن `record_id` يُصبح NULL لكن `submitted_at` يبقى — مما يُحافظ على سجل التاريخ ويمنع إعادة التعبئة الغير مقصودة.

### التعديلات على جدول `projects`

```sql
ALTER TABLE projects ADD COLUMN participants_enabled          BOOLEAN DEFAULT false;
ALTER TABLE projects ADD COLUMN participant_name_field        TEXT;
ALTER TABLE projects ADD COLUMN participant_edit_hours        INTEGER DEFAULT 48;
ALTER TABLE projects ADD COLUMN participant_allow_open        BOOLEAN DEFAULT false;
```

### حالات المشارك الثلاث

| الحالة | الشرط البرمجي | الأيقونة | ما يرى المشارك عند فتح رابطه | التذكير المناسب |
|---|---|---|---|---|
| **لم يفتح الرابط** | `first_opened_at IS NULL AND submitted_at IS NULL` | 🔴 | النموذج كاملاً للتعبئة | رسالة تنبيه |
| **فتح ولم يُكمل** | `first_opened_at IS NOT NULL AND submitted_at IS NULL` | 🟡 | النموذج كاملاً للتعبئة (يُستأنف) | رسالة تحفيزية |
| **عبّأ — ضمن فترة التعديل** | `submitted_at IS NOT NULL AND submitted_at > NOW() - INTERVAL '1 hour' * participant_edit_hours` | 🟢✏️ | النموذج قابل للتعديل |  — |
| **عبّأ — انتهت الفترة** | `submitted_at IS NOT NULL AND فترة التعديل انتهت` | 🟢🔒 | رسالة: "انتهت فترة التعديل، شكراً لك" | — |

## 5. نقاط API — مواصفة كاملة

### 5.1 إدارة المشاركين (للمدير)

#### `GET /api/projects/:id/participants`
**الوصف:** قائمة المشاركين مع حالة كل منهم

**Query params:**
- `status` — `all | submitted | opened | not_opened` (default: `all`)
- `search` — بحث نصي في الاسم والـ identifier
- `page`, `limit` (default: 50)

**الاستجابة — نجاح `200`:**
```json
{
  "participants": [
    {
      "id": "uuid",
      "name": "محمد الأحمد",
      "identifier": "@mohammed",
      "identifierType": "phone",
      "status": "submitted",
      "submittedAt": "2026-07-10T14:30:00Z",
      "firstOpenedAt": "2026-07-10T12:00:00Z",
      "notifyCount": 1,
      "recordId": "uuid"
    }
  ],
  "summary": { "total": 25, "submitted": 18, "opened": 3, "notOpened": 4 }
}
```

---

#### `POST /api/projects/:id/participants`
**الوصف:** إضافة مشارك + إرسال تيليغرام تلقائي

**Body:**
```json
{
  "name": "سارة المطيري",
  "identifier": "0509876543",
  "identifierType": "phone",
  "prefillData": { "department": "الموارد البشرية" },
  "notes": ""
}
```

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| نجاح | `201` | `{ "participant": {...}, "telegramSent": true }` |
| حقل إلزامي ناقص | `400` | `{ "error": "name مطلوب" }` |
| المشروع غير موجود | `404` | `{ "error": "المشروع غير موجود" }` |
| الميزة غير مُفعَّلة | `409` | `{ "error": "تتبع المشاركين غير مُفعَّل لهذا المشروع" }` |

---

#### `POST /api/projects/:id/participants/import`
**الوصف:** استيراد دفعي من Excel مع ربط الأعمدة

**Body (multipart/form-data):**
- `file` — ملف Excel
- `mapping` — JSON: `{ "اسم الموظف": "name", "رقم الجوال": "identifier", "القسم": "department" }`
- `identifierType` — `phone | email | null`
- `overwriteDuplicates` — `boolean` (كيفية التعامل مع التكرار — انظر §8)

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| نجاح جزئي أو كلي | `200` | `{ "imported": 23, "skipped": 2, "skippedReasons": [...] }` |
| ملف غير صالح | `400` | `{ "error": "صيغة الملف غير مدعومة" }` |
| العمود المُعيَّن للاسم فارغ | `400` | `{ "error": "عمود الاسم لا يمكن أن يكون فارغاً" }` |

---

#### `POST /api/projects/:id/participants/notify-pending`
**الوصف:** إرسال تذكيرات ذكية بشروط

**Body:**
```json
{
  "target": "not_opened | opened_only | all_pending",
  "minHoursSinceLastNotify": 24,
  "respectMaxLimit": true
}
```

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| نجاح | `200` | `{ "sent": 6, "skippedMaxLimit": 1, "skippedTooSoon": 2 }` |
| لا يوجد telegram_chat_id لأي مشارك | `200` | `{ "sent": 0, "reason": "لم يُفعِّل أي مشارك البوت بعد" }` |

---

#### `GET /api/projects/:id/participants/export`
**الوصف:** تصدير تقرير Excel

**الاستجابة — نجاح `200`:** ملف `.xlsx` مع رأس:
`Content-Disposition: attachment; filename="participants-{projectSlug}-{date}.xlsx"`

**أعمدة الملف:** الاسم | نوع التواصل | التواصل | الحالة | أول فتح | وقت التعبئة | عدد التذكيرات | ملاحظات

---

### 5.2 النموذج العام (للمشارك)

#### `GET /api/pform/:projectId/participant/:token`
**الوصف:** التحقق من الرابط + تسجيل `first_opened_at` + إعادة بيانات المشارك

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| رابط صالح لم يُعبَّأ | `200` | `{ "participant": { name, prefillData }, "alreadySubmitted": false, "canEdit": false }` |
| رابط صالح وضمن فترة التعديل | `200` | `{ "participant": {...}, "alreadySubmitted": true, "canEdit": true, "recordId": "uuid" }` |
| رابط صالح وانتهت فترة التعديل | `200` | `{ "participant": {...}, "alreadySubmitted": true, "canEdit": false }` |
| token غير موجود | `404` | `{ "error": "الرابط غير صالح" }` |
| token موجود لكن المشروع مُعطَّل | `403` | `{ "error": "النموذج مغلق حالياً" }` |

> **جانب الخادم:** إذا كان `first_opened_at IS NULL` يُعيَّن إلى `NOW()` عند هذا الطلب.

### 5.3 Webhook تيليغرام

#### `POST /api/pform/telegram-webhook`
**الوصف:** يستقبل `/start {participant_token}` من تيليغرام ويحفظ `chat_id`

**الأمان — إلزامي:**
يجب التحقق من صحة كل طلب وارد باستخدام `X-Telegram-Bot-Api-Secret-Token` header مقارنةً بالقيمة المُخزَّنة في إعدادات المشروع. أي طلب بدون هذا الـ header أو بقيمة خاطئة يُرفض بـ `403`.

```typescript
// مثال التحقق
const secretToken = req.headers['x-telegram-bot-api-secret-token'];
if (secretToken !== project.telegramWebhookSecret) {
  return res.status(403).json({ error: 'Forbidden' });
}
```

**منطق المعالجة:**
```
1. استخراج participant_token من نص /start
2. البحث عن المشارك بالـ token
3. إذا وُجد: UPDATE project_participants SET telegram_chat_id = {chat_id}
4. إرسال رسالة تأكيد: "تم تفعيل التذكيرات ✅"
5. إذا لم يُوجَد: إرسال "هذا الرابط غير صالح أو منتهي الصلاحية"
```

**الاستجابات للخادم:**
| الحالة | Status |
|---|---|
| تمت المعالجة | `200` |
| Token خاطئ أو غياب الـ Secret | `403` |

### 6.4 تجربة المشارك — جميع الحالات

**المسار:** `/r/{project-slug}/p/{participant-token}`

| الحالة | ما يراه المشارك |
|---|---|
| رابط صالح — لم يعبّئ بعد (سواء فتح سابقاً أم لا) | النموذج مع بياناته مُعبّأةً مسبقاً، بدون كود دعوة |
| رابط صالح — عبّأ وضمن فترة التعديل | النموذج بقيمه الحالية قابل للتعديل |
| رابط صالح — عبّأ وانتهت فترة التعديل | رسالة: "انتهت فترة التعديل، شكراً لك" |
| الرابط العام + الميزة مُفعَّلة + `participant_allow_open = false` | رسالة: "هذا النموذج مخصص للمدعوين فقط" |
| الرابط العام + الميزة مُفعَّلة + `participant_allow_open = true` | النموذج العادي كالمعتاد |
| token غير موجود في قاعدة البيانات | رسالة: "هذا الرابط غير صالح" |

## 7. الإشعارات عبر تيليغرام

### 7.1 آلية الربط مع المشارك الفرد

```
رابط تفعيل البوت المُرسَل للمشارك:
t.me/{BotName}?start={participant_token}

الخطوات:
1. المشارك يضغط الرابط → يفتح البوت ويضغط Start
2. البوت يستقبل /start {participant_token}
3. التحقق من Secret Token في الـ header
4. النظام يبحث عن المشارك بالـ token
5. يحفظ telegram_chat_id في project_participants
6. يُرسل للمشارك: "تم تفعيل التذكيرات ✅"
7. من الآن — كل تذكير يُرسَل مباشرة لهذا المشارك
```

### 7.2 قوالب الرسائل

**أ. الإشعار المبدئي (تلقائي فور الإضافة):**
```
مرحباً {name}،

تمت دعوتك لتعبئة نموذج "{project_name}".

رابطك الشخصي:
{personal_link}

لتفعيل التذكيرات عبر تيليغرام، افتح:
{telegram_bot_link}
```

**ب. تذكير المتأخرين:**
```
تذكير: {name}،

لم تُكمل تعبئة نموذج "{project_name}" بعد.

رابطك الشخصي:
{personal_link}
```

**ج. تأكيد التعبئة الناجحة:**
```
✅ شكراً {name}،
```

### قواعد التكرار عند الاستيراد

| الحالة | السلوك الافتراضي (`overwriteDuplicates: false`) |
|---|---|
| اسم موجود مسبقاً بنفس المشروع | يُتخطى ويُضاف إلى `skippedReasons` |
| اسم جديد | يُضاف كمشارك جديد |
| صف بعمود الاسم فارغ | يُتخطى دائماً مع إشارة تحذير |

> البيانات المُربطة بحقول النموذج (غير الاسم والتواصل) تُخزَّن في `prefill_data JSONB` وتُحقَن في النموذج عند فتح الرابط الشخصي.

## 11. الحالات الحدية (Edge Cases)

| الحالة | السلوك المطلوب |
|---|---|
| مشارك يُرسل النموذج من جهازين بنفس الوقت | `INSERT` مشروط: السجل الثاني يُرفض إذا كان `record_id IS NOT NULL`. يُستخدم DB-level lock أو `UPDATE ... WHERE record_id IS NULL` |
| حذف سجل مشارك بعد تعبئته | `record_id` يُصبح NULL (ON DELETE SET NULL) — لكن `submitted_at` يبقى محفوظاً. المشارك يرى "انتهت فترة التعديل" وليس "لم يعبّئ" |
| تعطّل تيليغرام عند الإضافة | تُسجَّل الإضافة بنجاح بغض النظر عن نتيجة تيليغرام. المدير يستطيع إعادة الإرسال يدوياً لاحقاً من الجدول |
| مشارك لم يُفعِّل البوت ومحاولة إرسال تذكير | يُرسل الـ endpoint `{ "sent": 0, "reason": "telegram_chat_id غير مُسجَّل" }` — لا خطأ، فقط تقرير |
| تغيير Bot Token في إعدادات المشروع | جميع `telegram_chat_id` المُخزَّنة تُصبح غير صالحة. يجب إضافة تحذير في واجهة الإعدادات: "تغيير الـ Bot Token يتطلب إعادة تفعيل البوت من جميع المشاركين" |
| استيراد Excel بـ 500+ مشارك | يُنفَّذ كـ background job — يُعيد الـ endpoint `{ "jobId": "uuid", "status": "processing" }` ويُعلم المدير عند الانتهاء |
| فتح رابط مشارك تم حذفه من القائمة | `404` — "هذا الرابط غير صالح" |
