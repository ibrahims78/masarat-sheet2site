## 1. نظرة عامة على الميزة

### الهدف
تمكين مدير المشروع من تحديد قائمة بالمشاركين المطلوب منهم تعبئة النموذج، ومتابعة حالة كل منهم في الوقت الفعلي، مع إرسال تذكيرات مستهدفة ذكية للمتأخرين عبر تيليغرام.

### الفجوة الحالية
النظام الحالي يعرف **من عبّأ فعلاً**، لكنه لا يعرف **من كان مفترضاً أن يعبّئ**. هذه الميزة تُغلق هذه الفجوة وتُتيح قياس نسبة الاستجابة الحقيقية.

### أمثلة حالات الاستخدام
- إدارة موارد بشرية تتابع من أكمل نموذج الإجازة من 150 موظفاً
- مدرسة تتابع من أعاد نموذج موافقة ولي الأمر من 200 طالب
- جهة حكومية تتابع إقرارات 80 موظفاً

## 2. آلية العمل الكاملة

```
1. المدير يُفعِّل "تتبع المشاركين" في إعدادات المشروع
          ↓
2. الرابط العام يُغلق تلقائياً لهذا المشروع فقط
   (يظهر لمن يفتحه: "هذا النموذج مخصص للمدعوين فقط")
   (خيار: السماح لغير المُسجَّلين يُفعَّل يدوياً عند الحاجة)
          ↓
3. المدير يُضيف المشاركين (يدوياً أو باستيراد Excel)
          ↓
4. النظام يُولّد token فريد لكل مشارك ويُرسل تيليغرام تلقائياً
   الرسالة تحتوي: الرابط الشخصي + رابط تفعيل البوت
          ↓
5. المشارك يفتح رابط البوت مرة واحدة → النظام يحفظ chat_id تلقائياً
          ↓
6. المشارك يفتح رابطه الشخصي → يُسجَّل first_opened_at
   النموذج يظهر مع بياناته مُعبّأةً مسبقاً (بدون كود دعوة)
          ↓
7. بعد التعبئة → يُربط السجل بالمشارك
   يمكن التعديل خلال participant_edit_hours
   بعدها: رسالة "انتهت فترة التعديل"
          ↓
8. المدير يُتابع لوحة التحكم: 3 حالات في الوقت الفعلي
   ويُرسل تذكيرات ذكية للمتأخرين
```

## 3. تصميم قاعدة البيانات

### الجدول الجديد: `project_participants`

```sql
CREATE TABLE project_participants (
  id                UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id        UUID        NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  name              TEXT        NOT NULL,
  identifier        TEXT,                      -- رقم جوال أو إيميل (اختياري)
  identifier_type   TEXT,                      -- 'phone' | 'email' | NULL
  token             UUID        NOT NULL UNIQUE DEFAULT gen_random_uuid(),
  telegram_chat_id  TEXT,                      -- يُحفظ بعد أن يفتح المشارك رابط البوت
  prefill_data      JSONB,                     -- بيانات التعبئة المسبقة من استيراد Excel
  record_id         UUID        REFERENCES project_records(id) ON DELETE SET NULL,
  submitted_at      TIMESTAMPTZ,
  first_opened_at   TIMESTAMPTZ,              -- أول مرة يفتح الرابط (لم يكمل بعد)
  last_notified_at  TIMESTAMPTZ,
  notify_count      INT         NOT NULL DEFAULT 0,
  added_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
  notes             TEXT
);

CREATE INDEX ON project_participants(project_id);
CREATE INDEX ON project_participants(token);
CREATE INDEX ON project_participants(record_id);
CREATE INDEX ON project_participants(telegram_chat_id);
```

### التعديلات على جدول `projects`

```sql
ALTER TABLE projects ADD COLUMN participants_enabled          BOOLEAN DEFAULT false;
ALTER TABLE projects ADD COLUMN participant_name_field        TEXT;       -- حقل الاسم للتعبئة المسبقة
ALTER TABLE projects ADD COLUMN participant_edit_hours        INTEGER DEFAULT 48;  -- مستقل عن edit_token_hours
ALTER TABLE projects ADD COLUMN participant_allow_open        BOOLEAN DEFAULT false; -- السماح لغير المُسجَّلين استثناءً
```

### حالات المشارك الثلاث

| الحالة | الشرط | الأيقونة | التذكير المناسب |
|---|---|---|---|
| **لم يفتح الرابط** | `first_opened_at IS NULL AND submitted_at IS NULL` | 🔴 | رسالة تنبيه |
| **فتح ولم يُكمل** | `first_opened_at IS NOT NULL AND submitted_at IS NULL` | 🟡 | رسالة تحفيزية |
| **عبّأ** | `submitted_at IS NOT NULL` | 🟢 | — |

## 4. نقاط API

### إدارة المشاركين (للمدير — محمي بـ requireAuth)

| الطريقة | المسار | الوصف |
|---|---|---|
| `GET` | `/api/projects/:id/participants` | قائمة المشاركين مع حالة كل منهم + فلترة بالحالة |
| `POST` | `/api/projects/:id/participants` | إضافة مشارك + إرسال تيليغرام تلقائي |
| `PATCH` | `/api/projects/:id/participants/:pid` | تعديل بيانات مشارك |
| `DELETE` | `/api/projects/:id/participants/:pid` | حذف مشارك |
| `POST` | `/api/projects/:id/participants/import` | استيراد من Excel مع ربط الأعمدة |
| `GET` | `/api/projects/:id/participants/export` | تصدير تقرير Excel |
| `POST` | `/api/projects/:id/participants/:pid/notify` | تذكير مشارك محدد |
| `POST` | `/api/projects/:id/participants/notify-pending` | تذكير ذكي بشروط |

### النموذج العام (للمشارك — غير محمي)

| الطريقة | المسار | الوصف |
|---|---|---|
| `GET` | `/api/pform/:projectId/participant/:token` | التحقق + تسجيل `first_opened_at` + إعادة بيانات التعبئة المسبقة |
| `POST` | `/api/pform/:projectId/submit` | يُرسل `participantToken` في الـ body — يتجاوز كود الدعوة |

### بوت تيليغرام (معالج الـ `/start`)

| الحدث | المسار | الوصف |
|---|---|---|
| `POST` | `/api/pform/telegram-webhook` | يستقبل `/start TOKEN` → يحفظ `telegram_chat_id` في `project_participants` |

## 5. الإشعارات عبر تيليغرام

### 5.1 آلية الربط مع المشارك الفرد

```
رابط تفعيل البوت المُرسَل للمشارك:
t.me/{BotName}?start={participant_token}

الخطوات:
1. المشارك يضغط الرابط → يفتح البوت ويضغط Start
2. البوت يستقبل /start {participant_token}
3. النظام يبحث عن المشارك بالـ token
4. يحفظ telegram_chat_id في project_participants
5. من الآن — كل تذكير يُرسَل مباشرة لهذا المشارك
```

> هذا الربط يحدث مرة واحدة فقط ويبقى طالما المشارك لم يُحذف.

### 5.2 قوالب الرسائل

**أ. رسالة الإشعار المبدئي (تُرسَل تلقائياً فور الإضافة):**
```
مرحباً {name}،

تمت دعوتك لتعبئة نموذج "{project_name}".

رابطك الشخصي:
{personal_link}

لتفعيل التذكيرات عبر تيليغرام، افتح:
{telegram_bot_link}
```

**ب. رسالة التذكير للمتأخرين:**
```
تذكير: {name}،

لم تُكمل تعبئة نموذج "{project_name}" بعد.

رابطك الشخصي:
{personal_link}
```

**ج. تأكيد الاستلام للمشارك (بعد التعبئة):**
```
✅ شكراً {name}،
تم استلام تعبئتك لنموذج "{project_name}" بنجاح.
يمكنك التعديل لمدة {participant_edit_hours} ساعة عبر نفس الرابط.
```

### 5.3 التذكيرات الذكية

```
واجهة "إرسال تذكير":
┌─────────────────────────────────────────────┐
│  إرسال تذكير لـ:                            │
│  ○ من لم يفتح الرابط بعد  (🔴 4 مشاركين)  │
│  ○ من فتح ولم يكمل        (🟡 3 مشاركين)  │
│  ○ الجميع المتأخرين        (7 مشاركين)     │
│                                             │
│  الشروط:                                    │
│  □ فقط من مضى على آخر تذكير > [24] ساعة   │
│  (الحد الأقصى: 3 تذكيرات — لن يُرسَل      │
│   لمن استنفد الحد)                          │
│                                             │
│  [إرسال]                                    │
└─────────────────────────────────────────────┘
```

> **قاعدة الحد الأقصى:** لا يُرسَل تذكير لمشارك وصل `notify_count` إلى 3، حتى لو اختار المدير "تذكير الجميع".

### 6.4 تجربة المشارك — رابطه الشخصي


|---|---|

## 8. QR Code وورقة الطباعة

### QR لكل مشارك (في الجدول)
- زر [QR] بجانب كل مشارك يفتح نافذة تعرض QR code لرابطه الشخصي
- يمكن تحميله كصورة PNG

### ورقة الطباعة الجماعية (PDF)
- زر "🖨️ طباعة ورقة الأدوار" يُولّد PDF
- كل مشارك في بطاقة: الاسم + QR code لرابطه
- مثالي للتوزيع في الاجتماعات والفصول الدراسية
