## 1. نظرة عامة على الميزة

### الهدف
تمكين مدير المشروع من تحديد قائمة بالمشاركين المطلوب منهم تعبئة النموذج، ومتابعة حالة كل منهم في الوقت الفعلي، مع إرسال تذكيرات مستهدفة ذكية للمتأخرين عبر تيليغرام.

### الفجوة الحالية
النظام الحالي يعرف **من عبّأ فعلاً**، لكنه لا يعرف **من كان مفترضاً أن يعبّئ**. هذه الميزة تُغلق هذه الفجوة وتُتيح قياس نسبة الاستجابة الحقيقية.

### أمثلة حالات الاستخدام
- إدارة موارد بشرية تتابع من أكمل نموذج الإجازة من 150 موظفاً
- مدرسة تتابع من أعاد نموذج موافقة ولي الأمر من 200 طالب
- جهة حكومية تتابع إقرارات 80 موظفاً

## 2. آلية العمل الكاملة

```
1. المدير يُفعِّل "تتبع المشاركين" في إعدادات المشروع
          ↓
2. الرابط العام يُغلق تلقائياً لهذا المشروع فقط
   (يظهر لمن يفتحه: "هذا النموذج مخصص للمدعوين فقط")
   (خيار: السماح لغير المُسجَّلين يُفعَّل يدوياً عند الحاجة)
          ↓
3. المدير يُضيف المشاركين (يدوياً أو باستيراد Excel)
          ↓
4. النظام يُولّد token فريد لكل مشارك
   إذا كان Bot Token مُعدَّاً → يُرسل تيليغرام تلقائياً فوراً
   إذا لم يكن مُعدَّاً → تُسجَّل الإضافة مع تحذير في الواجهة
   الرسالة تحتوي: الرابط الشخصي + رابط تفعيل البوت
          ↓
5. المشارك يفتح رابط البوت مرة واحدة → النظام يحفظ chat_id تلقائياً
          ↓
6. المشارك يفتح رابطه الشخصي → يُسجَّل first_opened_at
   النموذج يظهر مع بياناته مُقترَحةً مسبقاً (قابل للتعديل — بدون كود دعوة)
          ↓
7. بعد التعبئة → يُربط السجل بالمشارك
   يمكن التعديل خلال participant_edit_hours
   بعدها: رسالة "انتهت فترة التعديل، شكراً لك"
          ↓
8. المدير يُتابع لوحة التحكم: 4 حالات في الوقت الفعلي
   ويُرسل تذكيرات ذكية للمتأخرين
```

## 3. تصميم قاعدة البيانات

### الجدول الجديد: `project_participants`

```sql
CREATE TABLE project_participants (
  id                UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id        UUID        NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  name              TEXT        NOT NULL,
  identifier        TEXT,                      -- رقم جوال أو معرّف تيليغرام (اختياري)
  identifier_type   TEXT,                      -- 'phone' | 'telegram' | 'email' | NULL
  token             UUID        NOT NULL UNIQUE DEFAULT gen_random_uuid(),
  telegram_chat_id  TEXT,                      -- يُحفظ بعد أن يفتح المشارك رابط البوت
  prefill_data      JSONB,                     -- بيانات مُقترَحة للتعبئة من استيراد Excel
  record_id         UUID        REFERENCES project_records(id) ON DELETE SET NULL,
  submitted_at      TIMESTAMPTZ,               -- يُعيَّن عند ربط السجل — لا يُحذف عند حذف السجل
  first_opened_at   TIMESTAMPTZ,              -- أول مرة يفتح الرابط (لم يكمل بعد)
  last_notified_at  TIMESTAMPTZ,
  notify_count      INT         NOT NULL DEFAULT 0,
  added_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
  notes             TEXT
);

CREATE INDEX ON project_participants(project_id);
CREATE INDEX ON project_participants(token);
CREATE INDEX ON project_participants(record_id);
CREATE INDEX ON project_participants(telegram_chat_id);
```

> **ملاحظة:** `submitted_at` يُخزَّن مستقلاً عن `record_id`. إذا حُذف السجل لاحقاً فإن `record_id` يُصبح NULL لكن `submitted_at` يبقى — مما يُحافظ على سجل التاريخ ويمنع إعادة التعبئة غير المقصودة.

### التعديلات على جدول `projects`

```sql
ALTER TABLE projects ADD COLUMN participants_enabled          BOOLEAN DEFAULT false;
ALTER TABLE projects ADD COLUMN participant_name_field        TEXT;
ALTER TABLE projects ADD COLUMN participant_edit_hours        INTEGER DEFAULT 48;
ALTER TABLE projects ADD COLUMN participant_allow_open        BOOLEAN DEFAULT false;
ALTER TABLE projects ADD COLUMN telegram_webhook_secret      TEXT;  -- سر التحقق من هوية تيليغرام
```

### حالات المشارك الأربع

| الحالة | الشرط البرمجي | الأيقونة | ما يرى المشارك عند فتح رابطه | التذكير المناسب |
|---|---|---|---|---|
| **لم يفتح الرابط** | `first_opened_at IS NULL AND submitted_at IS NULL` | 🔴 | النموذج للتعبئة مع البيانات المقترحة | رسالة تنبيه |
| **فتح ولم يُكمل** | `first_opened_at IS NOT NULL AND submitted_at IS NULL` | 🟡 | النموذج للتعبئة (يُستأنف) | رسالة تحفيزية |
| **عبّأ — ضمن فترة التعديل** | `submitted_at IS NOT NULL AND submitted_at > NOW() - (participant_edit_hours \|\| ' hours')::interval` | 🟢✏️ | النموذج بقيمه الحالية قابل للتعديل | — |
| **عبّأ — انتهت الفترة** | `submitted_at IS NOT NULL AND submitted_at <= NOW() - (participant_edit_hours \|\| ' hours')::interval` | 🟢🔒 | رسالة: "انتهت فترة التعديل، شكراً لك" | — |

## 5. نقاط API — مواصفة كاملة

### 5.1 إدارة المشاركين (للمدير — محمي بـ requireAuth)

---

#### `GET /api/projects/:id/participants`
**الوصف:** قائمة المشاركين مع حالة كل منهم

**Query params:**
- `status` — `all | submitted | opened | not_opened` (default: `all`)
- `search` — بحث نصي في الاسم والـ identifier
- `page`, `limit` (default: 50)

**الاستجابة — نجاح `200`:**
```json
{
  "participants": [
    {
      "id": "uuid",
      "name": "محمد الأحمد",
      "identifier": "@mohammed_tg",
      "identifierType": "telegram",
      "status": "submitted",
      "submittedAt": "2026-07-10T14:30:00Z",
      "firstOpenedAt": "2026-07-10T12:00:00Z",
      "notifyCount": 1,
      "recordId": "uuid"
    }
  ],
  "summary": { "total": 25, "submitted": 18, "opened": 3, "notOpened": 4 }
}
```

---

#### `POST /api/projects/:id/participants`
**الوصف:** إضافة مشارك + محاولة إرسال تيليغرام

**Body:**
```json
{
  "name": "سارة المطيري",
  "identifier": "0509876543",
  "identifierType": "phone",
  "prefillData": { "department": "الموارد البشرية" },
  "notes": ""
}
```

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| نجاح + تيليغرام أُرسل | `201` | `{ "participant": {...}, "telegramSent": true }` |
| نجاح + تيليغرام لم يُرسَل (Bot غير مُعدَّ) | `201` | `{ "participant": {...}, "telegramSent": false, "telegramWarning": "Bot Token غير مُعدَّ في الإعدادات" }` |
| حقل إلزامي ناقص | `400` | `{ "error": "name مطلوب" }` |
| المشروع غير موجود | `404` | `{ "error": "المشروع غير موجود" }` |
| الميزة غير مُفعَّلة | `422` | `{ "error": "تتبع المشاركين غير مُفعَّل لهذا المشروع" }` |

---

#### `PATCH /api/projects/:id/participants/:pid`
**الوصف:** تعديل بيانات مشارك (الاسم / التواصل / الملاحظات / prefill_data)

**Body (جميع الحقول اختيارية):**
```json
{
  "name": "سارة المطيري الجديد",
  "identifier": "0509999999",
  "identifierType": "phone",
  "prefillData": { "department": "المالية" },
  "notes": "تذكير خاص"
}
```

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| نجاح | `200` | `{ "participant": { ...كامل البيانات المحدَّثة... } }` |
| المشارك غير موجود | `404` | `{ "error": "المشارك غير موجود" }` |
| لا يمكن تعديل `token` أو `telegram_chat_id` | `400` | `{ "error": "هذا الحقل لا يمكن تعديله يدوياً" }` |

> **ملاحظة:** تعديل `prefill_data` لا يؤثر على سجل مُعبَّأ مسبقاً — يؤثر فقط على التعبئة المسبقة للرابط عند الفتح التالي.

---

#### `DELETE /api/projects/:id/participants/:pid`
**الوصف:** حذف مشارك وفك ارتباطه بأي سجل

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| نجاح | `200` | `{ "deleted": true }` |
| المشارك غير موجود | `404` | `{ "error": "المشارك غير موجود" }` |
| المشارك عبّأ وضمن فترة التعديل | `200` | `{ "deleted": true, "warning": "تم الحذف — السجل المرتبط لا يزال موجوداً في قائمة السجلات" }` |

> **السلوك عند الحذف:** `record_id` في الجدول يُصبح NULL (ON DELETE SET NULL)، لكن السجل نفسه في `project_records` يبقى سليماً.

---

#### `POST /api/projects/:id/participants/:pid/notify`
**الوصف:** إرسال تذكير لمشارك واحد محدد

**Body (اختياري — لتخصيص الرسالة مستقبلاً):**
```json
{}
```

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| نجاح | `200` | `{ "sent": true, "notifyCount": 2 }` |
| المشارك لم يُفعِّل البوت | `200` | `{ "sent": false, "reason": "telegram_chat_id غير مُسجَّل بعد" }` |
| المشارك وصل الحد الأقصى (3) | `422` | `{ "error": "وصل المشارك الحد الأقصى للتذكيرات" }` |
| المشارك عبّأ بالفعل | `422` | `{ "error": "المشارك عبّأ النموذج — لا داعي للتذكير" }` |

---

#### `POST /api/projects/:id/participants/import`
**الوصف:** استيراد دفعي من Excel مع ربط الأعمدة

> **حد الأداء:** الاستيراد يُنفَّذ بشكل متزامن (synchronous) لحد أقصى **200 مشارك** في الطلب الواحد. لما فوق ذلك يُعاد `400` مع طلب تقسيم الملف — لا يوجد نظام background jobs في هذا الإصدار.

**Body (multipart/form-data):**
- `file` — ملف Excel
- `mapping` — JSON: `{ "اسم الموظف": "name", "رقم الجوال": "identifier", "القسم": "department" }`
- `identifierType` — `phone | telegram | email | null`
- `overwriteDuplicates` — `boolean`

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| نجاح جزئي أو كلي | `200` | `{ "imported": 23, "skipped": 2, "updated": 1, "skippedReasons": [...] }` |
| تجاوز الحد (200+) | `400` | `{ "error": "الحد الأقصى 200 مشارك لكل استيراد. قسّم الملف وأعد المحاولة." }` |
| ملف غير صالح | `400` | `{ "error": "صيغة الملف غير مدعومة" }` |
| عمود الاسم فارغ | `400` | `{ "error": "عمود الاسم لا يمكن أن يكون فارغاً" }` |

---

#### `POST /api/projects/:id/participants/notify-pending`
**الوصف:** إرسال تذكيرات ذكية بشروط

**Body:**
```json
{
  "target": "not_opened | opened_only | all_pending",
  "minHoursSinceLastNotify": 24,
  "respectMaxLimit": true
}
```

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| نجاح | `200` | `{ "sent": 6, "skippedMaxLimit": 1, "skippedTooSoon": 2, "skippedNoChatId": 3 }` |

---

#### `GET /api/projects/:id/participants/export`
**الوصف:** تصدير تقرير Excel

**الاستجابة — نجاح `200`:** ملف `.xlsx`
`Content-Disposition: attachment; filename="participants-{projectSlug}-{date}.xlsx"`

**أعمدة الملف:** الاسم | نوع التواصل | التواصل | الحالة | أول فتح | وقت التعبئة | عدد التذكيرات | ملاحظات

### 5.2 النموذج العام (للمشارك — غير محمي)

---

#### `GET /api/pform/:projectId/participant/:token`
**الوصف:** التحقق من الرابط + تسجيل `first_opened_at` + إعادة بيانات المشارك

**الاستجابات:**
| الحالة | Status | Body |
|---|---|---|
| لم يُعبَّأ بعد | `200` | `{ "participant": { name, prefillData }, "alreadySubmitted": false, "canEdit": false, "canSubmit": true }` |
| عبّأ + ضمن فترة التعديل | `200` | `{ "participant": {...}, "alreadySubmitted": true, "canEdit": true, "canSubmit": false, "recordId": "uuid" }` |
| عبّأ + انتهت الفترة | `200` | `{ "participant": {...}, "alreadySubmitted": true, "canEdit": false, "canSubmit": false }` |
| token غير موجود | `404` | `{ "error": "الرابط غير صالح" }` |
| المشروع مُعطَّل | `403` | `{ "error": "النموذج مغلق حالياً" }` |

> **حقلا `canSubmit` و `canEdit`:**
> - `canSubmit: true` — يظهر النموذج فارغاً أو بالبيانات المقترحة، قابل للإرسال
> - `canEdit: true` — يظهر النموذج بقيمه الحالية، قابل للتعديل
> - كلاهما `false` — يظهر للمشارك رسالة الاكتمال النهائية

> **جانب الخادم:** إذا كان `first_opened_at IS NULL` يُعيَّن إلى `NOW()` لدى هذا الطلب فقط.

---

#### `POST /api/pform/:projectId/submit`
**الوصف:** تعبئة النموذج من مشارك مُسجَّل

في الـ body يُرسَل `participantToken` → يتجاوز كود الدعوة → يُربط السجل الناتج بالمشارك تلقائياً ويُعيَّن `submitted_at`.

### 5.3 Webhook تيليغرام

---

#### `POST /api/pform/telegram-webhook`
**الوصف:** يستقبل جميع تحديثات البوت من تيليغرام — يُحال `/start {token}` لربط المشارك بالـ chat_id

**الأمان — إلزامي:**
الخطوة **الأولى** في المعالج (قبل أي منطق) هي التحقق من الـ header:
```typescript
const secretToken = req.headers['x-telegram-bot-api-secret-token'];
const expectedSecret = process.env.TELEGRAM_WEBHOOK_SECRET; // سر موحَّد على مستوى المنصة
if (!secretToken || secretToken !== expectedSecret) {
  return res.status(403).json({ error: 'Forbidden' });
}
```
> هذا الـ header يُرسَله **تيليغرام تلقائياً** عند كل update — ليس المشارك. يُعيَّن عند تسجيل الـ Webhook في تيليغرام مرة واحدة.

**نموذج الـ Webhook (مستوى المنصة لا المشروع):**
> البوت واحد على مستوى المنصة بـ `TELEGRAM_BOT_TOKEN` واحد → Webhook واحد موحَّد → المعالج يُحدد المشروع الصحيح عبر الـ `participant_token` الموجود في `/start`. لا تعارض بين المشاريع لأن الـ token فريد عالمياً (`UNIQUE` في قاعدة البيانات).

**منطق معالجة `/start`:**
```
1. استخراج participant_token من نص /start
2. البحث في project_participants عبر الـ token (UNIQUE)
3. إذا وُجد: UPDATE SET telegram_chat_id = {chat_id}
             إرسال: "✅ تم تفعيل التذكيرات بنجاح."
4. إذا لم يُوجَد: إرسال: "هذا الرابط غير صالح أو منتهي الصلاحية."
```

**الاستجابات:**
| الحالة | Status |
|---|---|
| تمت المعالجة | `200` |
| Secret خاطئ أو غائب | `403` |

### 6.5 شريط تحفيز تفعيل البوت (القرار 24)

عند فتح المشارك رابطه الشخصي — إذا كان `telegram_chat_id IS NULL` — يظهر شريط تحذير في أعلى الصفحة:

```
┌─────────────────────────────────────────────────────────┐
│  ⚠️  لتلقي التذكيرات على تيليغرام، فعّل البوت أولاً    │
│      [فتح البوت ←]                          [تجاهل ×]  │
└─────────────────────────────────────────────────────────┘
│                                                         │
│              [النموذج يظهر بالكامل هنا]                 │
│                                                         │
```

**السلوك:**
- الشريط يظهر فقط إذا لم يُفعَّل البوت بعد (`telegram_chat_id IS NULL`)
- زر "فتح البوت" يفتح `t.me/{BotName}?start={token}` في نافذة جديدة
- المشارك يستطيع تجاهله والتعبئة مباشرة — النموذج لا يُحجب
- بعد تفعيل البوت وإعادة فتح الرابط — الشريط يختفي تلقائياً

## 7. الإشعارات عبر تيليغرام

### 7.1 آلية الربط

```
رابط تفعيل البوت المُرسَل للمشارك:
t.me/{BotName}?start={participant_token}

ما يحدث:
1. المشارك يضغط الرابط → يفتح البوت ويضغط Start
2. تيليغرام يُرسل للـ Webhook: /start {participant_token}
3. المعالج يتحقق من X-Telegram-Bot-Api-Secret-Token (header تيليغرام)
4. يبحث عن المشارك بالـ token في قاعدة البيانات
5. يحفظ telegram_chat_id في project_participants
6. يُرسل للمشارك: "✅ تم تفعيل التذكيرات بنجاح."
   هذا يحدث مرة واحدة فقط ويبقى طالما المشارك لم يُحذف
```

### 7.2 قوالب الرسائل

**أ. الإشعار المبدئي (تلقائي فور الإضافة):**
```
مرحباً {name}،

تمت دعوتك لتعبئة نموذج "{project_name}".

رابطك الشخصي:
{personal_link}

لتفعيل التذكيرات عبر تيليغرام، افتح:
{telegram_bot_link}
```

**ب. تذكير المتأخرين:**
```
تذكير: {name}،

لم تُكمل تعبئة نموذج "{project_name}" بعد.

رابطك الشخصي:
{personal_link}
```

**ج. تأكيد التعبئة الناجحة:**
```
✅ شكراً {name}،
تم استلام تعبئتك لنموذج "{project_name}" بنجاح.
يمكنك التعديل لمدة {participant_edit_hours} ساعة عبر نفس الرابط.
```

**د. تأكيد تفعيل البوت:**
```
✅ تم تفعيل التذكيرات بنجاح.
ستصلك إشعارات نموذج "{project_name}" هنا مباشرة.
```

│  ○ من لم يفتح الرابط بعد  (🔴 4)           │

## 11. الحالات الحدية (Edge Cases)

| الحالة | السلوك المطلوب |
|---|---|
| مشارك يُرسل النموذج من جهازين بنفس الوقت | `UPDATE ... WHERE record_id IS NULL` — السجل الثاني يُرفض لأن `record_id` امتلأ بالأول |
| حذف سجل مشارك بعد تعبئته | `record_id = NULL` (ON DELETE SET NULL) — لكن `submitted_at` يبقى — المشارك يرى "انتهت فترة التعديل" وليس "لم يعبّئ" |
| تعطّل تيليغرام عند الإضافة | تُسجَّل الإضافة بنجاح — `telegramSent: false` في الاستجابة — المدير يُعيد الإرسال يدوياً لاحقاً |
| مشارك لم يُفعِّل البوت ومحاولة إرسال تذكير | `{ "sent": false, "reason": "telegram_chat_id غير مُسجَّل" }` — لا خطأ، فقط تقرير |
| تغيير Bot Token في إعدادات المنصة | الـ Webhook يُعاد تسجيله — جميع `telegram_chat_id` المُخزَّنة تبقى صالحة لأن الـ chat_id لا يتغير بتغيير البوت |
| استيراد Excel بأكثر من 200 مشارك | `400` مع رسالة تطلب تقسيم الملف — لا background jobs في هذا الإصدار |
| حذف مشارك وإعادة إضافته لاحقاً | token جديد يُولَّد — الرابط القديم يُصبح غير صالح تلقائياً |
| فتح الرابط العام و`participant_allow_open = true` | يُعبِّئ النموذج بدون تتبع — سجله في `project_records` بدون `participant_id` — لا يظهر في قائمة المشاركين |

## 12. ملاحظة أمان: مدة صلاحية الـ Token

الـ `token` في الإصدار الأول **لا ينتهي**. هذا قرار مقصود لتبسيط تجربة المشارك (لا حاجة لتجديد) مع القبول بالمخاطرة الصغيرة في حال تسريب الرابط.

**للإصدارات المستقبلية:** يمكن إضافة `token_expires_at TIMESTAMPTZ` وجعل الرابط ينتهي بعد تاريخ المشروع أو بعد تعبئة المشارك — هذا القرار مؤجَّل عمداً.
